Etiquetas

# Nombre Para qué sirve Aspecto
1 Head Contiene información de carácter general (metadatos) que no se muestra en la ventana del navegador. < head > < / head >
2 Title Contiene el título del documento. < title > < /title >
3 Link Enlazan a diferentes documentos o también hojas de estilo < link rel="stylesheet" href="estilo.css" >
4 Meta No tienen un propósito definido. Pueden contener cualquier tipo de información,
pero el agente que recibe la página web debe ser capaz de interpretar esa información. 		< meta charset="utf-8" >
5 Style Permite incluir en una página web declaraciones de estilo que se aplican al propio documento < style > < / style >
6 body Delimita el cuerpo del documento < body > < /body>
7 a Nos ayuda a poder crear un enlace a una página web < a > < /a >
8 Div Nos ayuda a estructurar el documento en secciones. < div > < /div >
9 Strong Es posible dar énfasis a un texto con esta etiqueta. < strong > < /strong >
10 br Con esta etiqueta hacemos salto de línea < br >
11 Img Añadimos alguna imagen al documento < img >
12 Ol Es para listas ordenadas < ol >
13 Span Se usa para estilos de una parte del texto < span > < /span >
14 P Se usa para organizar el texto < p > < /p >
15 Script Define ya sea un script interno o un enlace hacia un script externo. < script > < /script >
16 Noscript Define un contenido alternativo a mostrar cuando el navegador no soporta scripting. < noscript > < /noscript >
17 Section Sirve para determinar qué contenido corresponde a qué parte de un esquema. < section >
18 Header Define la cabecera de una página o sección < header >
19 Footer Representa un pie de página para el contenido de sección más cercano o el elemento raíz de sección < footer >
20 Address Define una sección que contiene información de contacto. < address >
21 Main Define el contenido principal o importante en el documento. < main >
22 Hr Representa un quiebre temático entre párrafos de una sección o articulo o cualquier contenido. < hr1 >...< /hr1>...< hr6 >...< /hr6>
23 Pre Indica que su contenido esta preformateado y que este formato debe ser preservado. < pre >
24 Blockquote Representa un contenido citado desde otra fuente. < blockquote > y < /blockquote >
25 ul Define una lista de artículos sin orden. < ul > < /ul >
26 Figure Representa una figura ilustrada como parte del documento. < figure >
27 li Define un artículo de una lista enumerada. < li > < /li>
28 dl Define una lista de definiciones, es decir, una lista de términos y sus definiciones asociadas. < dl > < /dl>
29 dt Representa un término definido por el siguiente < dt > < / dt>
30 dd Representa la definición de los términos listados antes que él. < dd > < /dd >
31 Figcaption Representa la leyenda de una figura. < figcaption >
32 em Representa un texto enfatizado, como un acento de intensidad. < em > < /em >
33 Small Representa un comentario aparte. < small > < /small >
34 s Representa contenido que ya no es exacto o relevante. < s > < /s >
35 Cite Representa el título de una obra. < cite > < /cite>
36 q Representa una cita textual inline. < q > < /q >
37 dfn Representa un término cuya definición está contenida en su contenido ancestro más próximo. < dfn > < /dfn >
38 Abbr Representa una abreviación o un acrónimo < abbr > < /abbr>
39 data Asocia un equivalente legible por máquina a sus contenidos. < data >
40 Code Representa un código de ordenador. < code > < /code >
41 i Representa un texto en una voz o estado de ánimo alterno. < i > < /i >
42 b Representa un texto hacia el cual se llama la atención para propósitos utilitarios. < b > < /b >
43 u Representa una anotación no textual sin-articular. < u > < /u >
44 br Representa un salto de línea. < br >
45 Table Representa datos con más de una dimensión. < table > < /table >
46 Label Representa el título de un control de formulario. < label > < /label >
47 Button Representa un botón. < button > < /button >
48 Menu Representa una lista de comandos . < menu > < /menu >
49 Legend Representa el título de un < fieldset >. < legend > < /legend >
50 Input Representa un campo de datos escrito que permite al usuario editar los datos. < input >
